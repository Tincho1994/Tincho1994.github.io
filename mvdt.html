Vibration Diagnostic Tool for Machine Health

Spring 2017
Embedded Systems Project

Previous Next

This project aimed to create a proof of concept for a portable machine health diagnostic tool as a capstone project for an embedded systems course. Faults developed over time in turbines and engines can be diagnosed using vibration analysis. This tool used an accelerometer, raspberry Pi, and display to allow a user to quickly characterize the vibrations of a machine.

The accelerometer communicated to the Pi through SPI and transmitted data at 25.6 KHz. This data was collected using a C script and written to a pipe. Meanwhile, a python script read the pipe, converted the measurements, and then performed a Fast Fourier Transform and output the result to a graph that was updated in real-time. By running data collection and processing as separate threads we were able to create a system that could reliably gather data at a fast rate.
